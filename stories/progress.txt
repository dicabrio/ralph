## Codebase Patterns (Updated by Agents)
- TanStack Start with file-based routing in src/routes/
- Component files use PascalCase (e.g., Header.tsx)
- TailwindCSS v4 via @tailwindcss/vite plugin
- Path alias @ configured for src/ folder in vite.config.ts
- Lucide React for icons
- Root layout in src/routes/__root.tsx with RootDocument shell component
- Header component contains sidebar navigation with slide-out menu
- Dev server runs on port 9000
- Project uses pnpm (v8.x) with lockfile v6 - use pnpm@8.15.9 for Docker builds
- Docker build uses multi-stage build: deps → builder → runner
- TanStack Start builds to .output/ folder, run with `node .output/server/index.mjs`
- Docker health checks use 127.0.0.1 (not localhost) to avoid IPv6 issues in Alpine
- Docker socket mount pattern for Docker-out-of-Docker (DooD) to spawn Claude containers
- Helper scripts in scripts/ folder for Claude container management
- HOST_PROJECTS_ROOT and HOST_SKILLS_PATH env vars needed for mounting host paths in spawned containers

**Learnings:**
- Docker-out-of-Docker (DooD) is simpler and safer than Docker-in-Docker (DinD)
- Docker socket mount allows spawning containers on the host from within a container
- Host paths must be passed explicitly (HOST_PROJECTS_ROOT) because container paths differ from host paths
- Alpine's docker-cli package is lightweight (~15MB) and sufficient for container management
- Using profiles: [manual] in docker-compose prevents a service from starting with `docker-compose up`
